Installation

HoldOn is a jQuery plugin but the overlay will be handled internally by the plugin, so we will call HoldOn.js with a variable.

<head>
    <meta charset="utf-8" />
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8"/>
    <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1.0"/>
    <title>HoldOn.js Demo</title>

    <!-- Load the css and js file in your document -->
    <link href="path-to/HoldOn.min.css" rel="stylesheet">
    <script src="path-to/HoldOn.min.js"></script>
</head>

Then a global variable HoldOn will be accesible on your document and all the features that HoldOn have to offer will be available for its use.

Show HoldOn.js

To lock the ui, we will use the HoldOn.open function, this function is ready to use but you can customize it how you want with javascript. 

var options = {
    message:"Welcome to the jungle babe"
};

HoldOn.open(options);

// or well without options to simply show a load div
// HoldOn.open();

Read the documentation to see more available options like change the font color, background color etc.
Remember that only 1 instance of HoldOn can be opened at once, if you execute a HoldOn.open when an instance is already running, this will be replaced with the new one.

Close HoldOn.js

To close an active instance of HoldOn, we will use the HoldOn.close function, this will close HoldOn overlay allowing again the user interaction. 

// Really easy isn't?
HoldOn.close();

About HoldOn.js

Humans are stupid by nature, all of us. As a developer, the things that make you most angry is a report of bugs in your code right? Nothing to do, you must write better code. However there are some situations in which is there so many logic that you need to execute to prevent that the user do an action when something important is running in the background. Horrible isn't, I wouldn't write so much code to prevent that? To make the things easier for you and me, HoldOn.js will simply show a loading div to prevent that the user do something stupid while this task is accomplished.
Very simple but functional.
